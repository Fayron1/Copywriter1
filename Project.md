Проект: Автономная мультиагентная SEO/GEO редакция «Антигравити»
1. Суть и миссия проекта
Глобальная цель: Создание полностью автономной ИИ-системы генерации контента для бизнеса (В2В, юриспруденция, налоги, экономика), которая превосходит по качеству тексты живых Senior-копирайтеров.
Главный враг проекта: «Синтетический» текст, вода, галлюцинации и типичные ИИ-клише. Статьи должны быть экспертными, опираться на твердые факты и легко читаться.
Среда разработки: Проект разворачивается в инфраструктуре «Антигравити» (Google).

2. Технический стек и роли ИИ-моделей
В проекте используется гибридная архитектура для достижения максимальной эффективности:

Claude Opus 4.6 (Ты — Архитектор): Твоя задача — писать Python-скрипты, настраивать пайплайны, проектировать базу данных и создавать филигранные системные промпты для GPT-агентов. Ты не пишешь сами статьи, ты создаешь завод по их производству.

GPT-5.x (Рабочая группа): Выполняют роли специализированных агентов (Писатель, Редактор, SEOшник и т.д.) через вызовы по единому OpenAI API ключу.

Gemini 3.1 (Frontend-разработчик): Отвечает за генерацию HTML/CSS/JS интерфейса админ-панели (Vibe Coding).

База данных: Qdrant (Векторная база для RAG-архитектуры).

Изображения: GPT Image 2.0 (DALL-E 4).

3. Типы контента (Форматы)
Система должна уметь генерировать статьи по жестким форматам:

Чек-листы (10 пунктов): Структура «Текст + Картинка» для каждого пункта.

Ситуации (Кейсы): Разбор проблемы, аналитика, выводы.

Разбор законодательства: Перевод нормативно-правовых актов (НПА) на простой и понятный для бизнеса язык.

Полезное (Памятки/Таблицы): Справочная информация, структурированная в Markdown-таблицы.

Ошибки (Анти-паттерны): Разбор того, «как делать нельзя».

4. Мультиагентная структура (GPT Agents)
Ты должен настроить взаимодействие следующих узкоспециализированных агентов:

Оркестратор MCP (The Brain): Декомпозирует задачу, управляет контекстом и маршрутизирует процессы между остальными агентами.

Исследователь + Юрист (The Fact-Finder): Работает с Qdrant. Ищет номера законов, экономические выкладки и судебную практику. Не выдумывает факты.

Анализатор трендов (The Scout): Парсит веб (Live Search). Ищет инфоповоды и актуальные новости для привязки сухой темы к текущему дню.

Структурировщик (The Engineer): Собирает жесткий логический каркас (H1-H3). Расставляет факты по пирамиде Минто. Оставляет теги для таблиц и картинок.

Писатель (The Heart): Пишет текст в стиле «Пиши, сокращай» и правилах Норы Галь. Создает динамичный ритм (Burstiness), избегает канцелярита.

Редактор (The Sheriff): Проверяет текст Писателя. Контролирует «Индекс Синтетичности» (отбраковывает текст с клише и водой) и проводит фактчекинг. Возвращает на доработку при необходимости.

SEO & GEO Специалист (The Booster): Внедряет LSI-семантику, создает «Наживки для цитирования» (прямые ответы для ИИ-поисковиков), пишет Alt-тексты и Schema.org разметку.

Визуализатор (The Artist): Создает точные промпты для GPT Image 2.0 с учетом единого брендбука (Seed) и формирует изображения для чек-листов.

5. База Знаний (Knowledge Base & Qdrant)
Для обеспечения экспертности (RAG) система использует векторную базу Qdrant.

Источники данных: Более 140 файлов (актуальные законы РФ на 2026 год, книги по экономике, маркетингу, сторителлингу и редактуре).

Скрипт парсинга: Скрипт-загрузчик должен корректно обрабатывать форматы FB2 (парсинг по XML-тегам <p>, <title>) и PDF. FB2 является приоритетным форматом для сохранения семантической структуры абзацев.

Метаданные (Payload): Чанки в базе должны иметь строгие теги (expertise_level, source_type, valid_until для законов).

6. Пост-обработка и SEO/GEO Скрипты
Ты должен написать Python-скрипт (Image Processor), который будет обрабатывать картинки перед публикацией на сайт:

Очистка: Удаление следов генерации ИИ (EXIF/IPTC).

Мимикрия: Программная замена тега Software на Adobe Photoshop 2026 (Windows).

Инъекция SEO/GEO: Внедрение ключевых слов (Keywords), описаний (Alt-text) и GPS-координат клиента (широта/долгота) напрямую в метаданные изображения для локального ранжирования.

Оптимизация: Сохранение финального результата в формате WebP.

Общая концепция
Создание универсального бота-копирайтера, который сможет создавать различный контент в соответствии с заданными форматами (категориями).

Функционал бота-копирайтера
Типовые форматы: Контент будет разделен на категории (чек-листы, новости, справки по определенным темам, памятки в виде таблиц, материалы "как делать нельзя" и т.д.).

Гибкие настройки: У каждой категории будет свой заданный промпт (шаблон запроса), количество знаков, смысл и периодичность публикаций.

Администрирование:

Возможность добавления новых категорий и редактирования промптов (для корректировки ответов бота).

Возможность задания периодичности публикации (сколько материалов в месяц).

Роль бота-модератора
Контроль: Бот-модератор должен корректировать то, что пишет бот-копирайтер.

Задачи:

Проверка грамматики, орфографии, актуальности данных и смыслового наполнения.

Использование собственного универсального промпта для проверки определенных моментов.

Соблюдение жесткого контроля качества.

Архитектура и интеграция

Связь с сайтами: Автоматическая интеграция с одним или несколькими сайтами. Материалы генерируются на стороне админки, а затем автоматически публикуются на соответствующие ресурсы.

Третий агент (поисковик): Планируется создание третьего агента, отвечающего за поиск актуальных тем в поисковых системах. Цель — адаптация контента под высокочастотные запросы и повышение посещаемости.
