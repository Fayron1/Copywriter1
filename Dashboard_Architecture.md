# Архитектура Дашборда "Antigravity Command Center"

## 1. Концепция и Роль
Дашборд — это пульт управления автономной ИИ-редакцией. Поскольку система уже умеет сохранять готовые статьи в виде структурированных файловых пакетов (`article.md`, `seo_package.json`, `pipeline_debug.json`), главной задачей дашборда становится **управление очередью генерации** и **дистрибуция артефактов**. 

Это не классическая CMS вроде WordPress, это терминал оператора контент-завода.

---

## 2. Дизайн-система (Aesthetic Direction)
Опираясь на правила из `.agent/skills/frontend-design`, мы отказываемся от скучных "SaaS-шаблонов" (дефолтных теней, скруглений и шрифтов вроде Inter). 

*   **Направление:** *Industrial Utilitarian* (Индустриальный утилитаризм). Интерфейс должен выглядеть как профессиональный софт (например, терминал Bloomberg или среда разработки).
*   **Цветовая палитра:** Темная тема (Slate/Gunmetal из `ui-designer`), высококонтрастные неоновые акценты (Кислотно-зеленый для статуса "Опубликовано", Сигнально-оранжевый для "Turing Score Warning").
*   **Типографика:** Моноширинный шрифт (JetBrains Mono или Space Mono) для таблиц, логов и JSON, плюс жесткий гротеск (Space Grotesk) для интерфейса.
*   **Пространство:** Видимые границы блоков (1px solid borders), отсутствие размытых теней, компактное расположение элементов для работы с большими объемами данных.
*   **Стек:** Tailwind CSS (согласно `ui-designer`), кастомизированный под нашу палитру, React/Next.js.

## 3. Структура Модулей Дашборда

### Модуль 1: Матрица Генерации (Task Matrix)
Место, где оператор ставит задачи.
*   Форма запуска: Ввод темы, выбор из 5 типов статей (checklist, analysis и т.д.), ползунок объема.
*   Очередь задач: Таблица со статусами запущенных скриптов (в духе терминальных логов: `INFO | 📄 Статья...`).

### Модуль 2: Хранилище Артефактов (Output Explorer)
Поскольку система выдает пакеты данных в папку `output/`, дашборд должен их парсить и визуализировать.
*   Грид сгенерированных пакетов.
*   **Детальный просмотр пакета (Split-screen):**
    *   *Вкладка 1 (Markdown):* Отрендеренный текст статьи `article.md` с подсветкой маркеров картинок.
    *   *Вкладка 2 (SEO & GEO):* Визуализация `seo_package.json` (превью сниппета в Google, Citation Baits, метатеги).
    *   *Вкладка 3 (Debug & Schema):* Вывод `pipeline_debug.json` (Turing Score от Шерифа, AI-Score от Зеркала) и `schema.json`.
*   **Кнопки действий:** "Отправить в блог (API)", "Скачать ZIP", "Удалить".

### Модуль 3: Лаборатория Агентов (Prompt Engineering)
Интерфейс для тонкой настройки системы без залезания в код (`prompts_part1.py` и `part2.py`).
*   Карточки 9 агентов с возможностью быстрого редактирования их системных инструкций.
*   Управление глобальными переменными: порог Turing Score, лимиты символов для форматов.

### Модуль 4: Векторный Хаб (Knowledge Base)
Мониторинг базы знаний, к которой обращается Исследователь (Fact-Finder).
*   Статус Qdrant.
*   Интерфейс для загрузки новых PDF/FB2 книг и свежих законов.

## 4. Дифференциатор (Отличие от дефолтных UI)
Этот интерфейс избегает "generic UI", полностью отказываясь от карточного дизайна с мягкими тенями. Вместо этого он предлагает плотную информационную панель, где логи генерации, JSON-артефакты и отрендеренный Markdown сосуществуют в жесткой асимметричной сетке, подчеркивая, что это профессиональный инструмент для массового производства контента.
