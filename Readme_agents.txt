Технический совет по загрузке для Исследователя:
При обработке этих материалов вашим скриптом, настройте метаданные в Qdrant так:

source_priority: Законам ставьте «1» (наивысший), книгам — «2». Если информация в книге противоречит закону, Исследователь должен верить закону.

effective_date: Для законов обязательно укажите дату последней редакции. В 2026 году это критично, так как старые чанки из 2024 года могут безнадежно устареть.

chunk_type: Разделяйте «Определение», «Санкция/Штраф», «Инструкция».

Технический совет по загрузке для Структурировщика:
При разбиении на чанки для этого агента в Qdrant, используйте следующие метаданные:

logical_pattern: Тегируйте чанки по типам структур (например: problem-solution, pyramid-principle, comparison-table, storytelling-loop). Это позволит Оркестратору MCP подавать Структурировщику нужные инструкции в зависимости от выбранного формата статьи.

attention_level: Указывайте, для какой части текста предназначен чанк (intro, body, conclusion, call-to-action).

visual_anchor: Для чек-листов помечайте чанки, которые содержат правила связки «Текст + Картинка», чтобы Структурировщик оставлял «якоря» для будущего Визуализатора.

Технический совет по загрузке для Писателя:
При индексации в Qdrant добавьте следующие метаданные к чанкам:

rhythm_type: помечайте чанки по длине и динамике предложений (short-punchy, balanced, explanatory). Это поможет Писателю менять темп текста.

emotive_charge: уровень эмоциональности (rational, inspiring, warning). Для категории «Ошибки» пригодится warning, для «Чек-листов» — rational.

voice_key: если у вас несколько сайтов с разным стилем, добавьте ключи (professional, startup-like, academic).

Технический совет по загрузке для Редактора:
Для этого агента в Qdrant критически важно использовать тег expertise_level: "ultra-expert". Редактор должен быть «умнее» и жестче Писателя.

Добавьте следующие метаданные к чанкам:

error_type: тегируйте ошибки (pleonasm, cliché, logic_gap, ai_marker). Это позволит Редактору быстро находить инструкции по исправлению конкретных дефектов.

cliché_replacement: создайте пары «штамп — живая замена». Например: «в настоящее время» → «сейчас» или «сегодня».

fact_check_protocol: алгоритм действий при обнаружении цифр или юридических терминов (обязательный запрос к Исследователю).

Технический совет по загрузке для SEO-оптимизатора:
Для этого агента в Qdrant добавьте следующие метаданные к чанкам:

intent_type: тегируйте чанки по целям пользователя (informational, transactional, navigational). Это поможет агенту выбирать правильные ключевые слова.

html_tag: помечайте правила оформления для разных элементов (title_tag, meta_description, h1_logic, faq_structure).

platform_spec: если у ваших 2-3 сайтов разные требования к SEO (например, один — блог, другой — база знаний), добавьте соответствующие ключи для фильтрации правил.

В вашей админке должен быть отдельный переключатель или блок настроек «GEO-акцент». Если он включен, агент-редактор будет проверять статью не на количество ключевых слов, а на количество «фактов на 1000 знаков» и наличие «блока прямого ответа».
